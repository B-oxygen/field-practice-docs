# 디슬롭 — 보고서 AI 문체 제거

생성된 활동보고서가 "AI가 찍어낸" 티가 날 때, 사실을 보존하며 사람 글처럼 다듬는다.

## AI 티 (정량 신호)
`python scripts/deslop_analyze.py report.hwpx` 로 측정(`--json` = 전체 리포트). 숫자·날짜·영문·인용은 **점수 전 마스킹**되어 사실은 절대 플래그 안 됨. 끝에 `risk_band`(low/medium/high) 산출.

도메인 신호(음슴체 활동보고서):
- **종결어미 단일성**: 활동문장이 거의 100% "~함."으로 끝남(가장 큰 신호).
- **동사 단조**: 정리/반영/확인 같은 소수 동사가 과반.
- **상투 마무리구**: "~체크리스트에 반영함" 류 반복.
- **필러**: "짧은 근무시간에 맞추어", 빈 "~관점에서/~기준으로".
- **경직된 구조**: 모든 항목이 `[제목]+불릿 3개`로 동일.

한국어 번역투/AI 문체 신호 (epoko77-ai/im-not-ai, MIT 차용):
- **번역투**: 에 대해 / 통해 / 위해 / 에 있어 / 라는 점에서 / 바탕으로 / 에 의해 남발.
- **이중피동**(되어진·보여진·쓰여진), **결론 피벗 ≥3**(따라서·결론적으로·종합하면), **과장어**(혁신적·획기적·압도적) → S1. 과장어는 실습보고서엔 특히 부적합.
- **가지고있다류**(have/make 직역), **접속사 남발**(또한·나아가·게다가).
- **어미 뒤 쉼표 ≥6**, **이중조사 >3**(에서의·으로의), **한자명사화(성/적/화) >12** → S2.

## 방법론 (공식문서 = 사실 보존 최우선)
1. **분석**: `deslop_analyze.py`로 현 상태 수치화.
2. **재작성**: 문장별 1:1, index 정렬로 재작성(음슴체 격식 유지).
   - 종결어미 자연 혼용, 필러 제거, 명사 나열(·) 축소, 동의어로 동사 분산.
   - **사실(영문·약어·숫자·고유명사) 100% 보존. 없는 사실 추가 금지.**
   - 분량 큰 경우 writing 에이전트에 위임하되 규칙을 엄격히.
3. **사실보존 게이트(필수)**: 
   ```bash
   python scripts/verify_fact_preservation.py pairs.tsv --proper-nouns "회사명,지역명"
   # pairs.tsv = old<TAB>new (문장별). 영문/숫자 토큰·고유명사 누락 0 이어야 통과.
   ```
4. **전수 정독**: 자동 게이트는 토큰만 본다. 한국어 뉘앙스·환각은 사람이 읽어 확인.
5. **적용**: `scripts/hwpx_replace.py`로 사본에 1:1 치환 → 재패키징.
6. **시각 QA**: 한글/렌더로 레이아웃 확인.

## 함정
- **치환 placeholder substring 충돌**: 두 토큰이 서로의 substring이면 "X not in" 검증이 깨진다. 비-substring 값 사용.
- **얕은 디슬롭**: 어미만 바꾸면 구조·내용 arc가 그대로라 여전히 AI 티. 구조까지 손대되 사실은 불변.
- **새 단조 유발**: 한 동의어(예: "취합")로 몰리면 새 편중. 분포를 다시 확인(`deslop_analyze.py`).
- 공식 제출문서는 **사실 정확 > 문체 완성**. 위험하면 보수적으로.
