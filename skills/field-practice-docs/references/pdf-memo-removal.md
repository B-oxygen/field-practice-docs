# PDF 메모(코멘트) 제거

한컴/HOP에서 내보낸 제출 PDF에 검토용 메모(초록 "USER" 박스)가 본문에 박혀 나오는 경우가 많다. 제출 전 제거.

## 핵심 판별
- 한글 메모는 PDF **주석(annotation)이 아니라 본문에 flatten**되는 경우가 흔하다 → `page.annots()`가 비어 있어도 텍스트로 존재.
- 확인: `page.get_text()`에 메모 문구가 있고, `page.search_for("문구")`로 위치가 잡히면 flatten된 것.

## 제거 전략 (`scripts/pdf_remove_memos.py`)
```bash
uv run --with pymupdf python scripts/pdf_remove_memos.py 입력.pdf 또는_폴더 출력폴더
# 옵션: --min-x-frac 0.72 (우측여백 기준), --pad 5
```
동작:
1. PDF 주석이 있으면 모두 삭제.
2. 초록 fill 드로잉(`fill≈(0.8,1.0,0.6)`)이면서 **우측여백(x0 ≥ width×0.72)** 에 있는 박스를 메모로 탐지.
3. 각 박스를 redact(텍스트 제거) + 흰색으로 덮음(초록/테두리 제거).
4. NFC 파일명으로 출력.

## 폼 콘텐츠 보존
- 폼 내부 초록 하이라이트(예: 날짜 강조)는 보통 x<좌측이라 `min-x-frac` 필터로 자동 보존된다.
- 색/위치가 다르면 `--min-x-frac` 조정. 메모만 정확히 잡히는지 먼저 1장으로 확인.

## 검증 (필수)
- 텍스트: 정리본에 메모 문구(예: "서명필수", "작성해주세요")가 0건인지 `get_text()`로 확인.
- 폼 핵심(표·날짜·서명·귀하)이 그대로인지 확인.
- **시각 QA**: `page.get_pixmap(dpi=110).save(png)` 후 렌더 이미지를 직접 보고 메모가 사라지고 잔여 흔적이 없는지 확인.
- 전수 처리 시 **출력 파일 개수·고유 파일명**을 확인(파일명 파싱 실패로 덮어쓰기 충돌 주의).

## 패키징
